# 2025年活動まとめ記事：制作プロセスの記録

この記事および集計データがどのように作成されたか、その技術的なプロセスを記録として残します。

## 1. 企画とセットアップ
- **目的:** GitHubユーザー `TsukemonoGit` の2025年の活動を詳細に数値化し、読み物としてまとめる。
- **環境準備:** GitHub Personal Access Token を `.env` ファイルに管理し、セキュアにAPIを叩く準備を実施。

## 2. データ取得（Pivoting）
- **初期アプローチ:** 当初は Python を使用してデータ取得を試みましたが、実行環境の制約により Node.js へシフト。
- **技術要素:** GitHub GraphQL API を使用。
- **取得内容:** カレンダーデータ（コントリビューション数）、リポジトリごとのコミット数、プルリクエスト数、Issue数などを、2025年1月1日から12月31日の範囲で取得。
- **成果物:** `activity_data.json`（生データ）

## 3. データの集計と分析
- **処理:** Node.js スクリプト（`summarize_activity.js`）を作成。
- **集計項目:** 
    - 年間総計
    - リポジトリ別ランキング
    - 月別の活動トレンド
- **成果物:** `summary_2025.md`（統計レポート）

## 4. 記事の執筆プロセス（反復的改善）
データから読み取れる「事実」に基づき、以下のステップで記事をブラッシュアップしました。
1. **ドラフト作成:** 統計データをベースに、英語でのナラティブ（物語）を作成。
2. **ローカライゼーション:** 日本語に翻訳し、ユーザー本人の視点（「私が書いた」の体裁）に調整。
3. **内容の拡充:** 
    - `lumilumi` への偏愛度の分析（コミット割合の計算）。
    - 7月・8月の「爆速開発モード」など、データ上の特異点を特定。
    - 技術スタック（Svelte/TypeScript）への言及を追加。
4. **最終化:** 読者（コミュニティ）に向けたメッセージを加え、熱量の高い詳細版（`Full`版）を完成。

## 5. 最終検証
- **期間検証:** 生データのタイムスタンプを確認し、1月1日から12月31日（未来分は0としてカウント）まで全期間が網羅されていることをダブルチェック。
- **整合性:** 記事内の数字と、APIから取得した数値に相違がないことを確認。
