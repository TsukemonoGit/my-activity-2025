# AIへの指示書（プロンプト）：2025年活動まとめを作る方法

この記事やシステムをAIにお願いして作ってもらうための「魔法の言葉（プロンプト）」をまとめました。
これをAIエージェント（Antigravity, Cursor, Bolt.dev, GitHub Copilotなど）に渡せば、誰でも自分のリポジトリの総括が作れます！

## 1. 最初のエンジニアリング指示：生データを取得する
まずは、AIにプログラムを書いてもらい、自分のGitHubから活動実績の生データを引っこ抜きます。

> **プロンプト例：**
> 
> [!IMPORTANT]
> この作業には **GitHub Personal Access Token** が必須です。
> トークンがない場合、API の制限でデータが正確に取得できなかったり（GraphQL API はトークンなしでは実行できません）、通常の API でも 100 件程度しか取得できなかったりします。必ず自分の開発用トークンを用意しましょう。
> 
> 「Githubのユーザー [自分のユーザー名] の2025年の活動履歴をまとめたい。
> まず、GitHubのGraphQL APIを使って、2025年1月1日から12月31日までのコントリビューション統計（コミット数、PR数、Issue数、リポジトリごとの内訳、月別の活動推移）を取得して、`activity_data.json`として保存するスクリプトを書いて。
> 言語はNode.js（または得意な言語）で。GitHubのトークンは `.env` から読み込むようにして。」

## 2. 記事の執筆指示：JSONを読み物に変換する
データが取得できたら、その `activity_data.json` をAIに読み込ませて、人間らしい記事に仕立ててもらいます。

> **プロンプト例：**
> 「保存した `activity_data.json` の中身を詳しく分析して、2025年の活動を振り返るまとめ記事を書いて。
> - 言語は日本語で。
> - 私が自分で書いたような（一人称）体裁にして。
> - 単なる数字の羅列じゃなく、どのプロジェクトを一番頑張ったか、どの時期が忙しかったか、どんな技術を使っているかなどの洞察を盛り込んだ、熱量の高いエッセイ風にして。
> - 内容をもりもりにして、読んだ人が驚くような充実した記事にして！」

## 3. 仕上げの指示
最後に、今回の私のような「メイキング」を作ってもらいます。

> **プロンプト例：**
> 「これまでの作業（データ取得から記事執筆まで）の流れを、技術的な記録として `creation_process.md` にまとめて。」

---

### 🔥 AIにうまく動いてもらうコツ
- **「事実に基づかせる」:** AIに「データ (`activity_data.json`) をしっかり見て書いて」と念を押し、嘘（ハルシネーション）を防ぐのがコツです。
- **「環境を合わせる」:** 「PythonがだめならNode.jsでやって」のように、自分のPCで動くものを柔軟に選んでもらうとスムーズです。
- **「追加分析をお願いする」:** 今回のように「7月・8月のピークについてもっと詳しく」などと追加で聞くと、より深い分析が出てきます。
